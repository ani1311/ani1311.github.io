Decentralized (Tree) Index Of The Internet

by Aniket Alur

November 23, 2021
Okay, what is the internet? essentially just a big graph of webpages. So is there be an easy way to see the entier internet. Now there are two challenges in this.

=> Who will run all that compute of crawllers.
=> Where to store graph.

The Idea:

A network where each node explores/updates some part of the internet. These nodes run crawllers and store the result of these crawllers, i.e some portion of the indexes of the internet on their network storage.
Now it is not possible to have the entire index of the internet stored on each node. So we can have a merkle tree like structure that each node stores and this structure is updated at each run of the node.

Why do this?

Well, for starters, if I want to scrape the web for data, this would be a fantastic resource. But more importantly, I don't know how or where someone might come up with an application that uses this API. But an index of the Internet sounds like a good resource. In addition, if someone can build a good parser, identifier, then this can server as a great tool to build new pages. We have hackerNews, but this might help build AritstNews or paleontologistNews. I think a publicly available index of the internet opens avenues that we might trancend the current internet ecosystem and take us to a new horizon.
